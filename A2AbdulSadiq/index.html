In case you use the command line:

1. Create a Repository and Push Local Repository on GitHub:

# Step 1: Initialize a local Git repository
git init

# Step 2: Add files and commit changes locally
git add .
git commit -m "Initial commit"

# Step 3: Create a new repository on GitHub (assuming you have already created a repo on GitHub)
# Follow GitHub's instructions for existing repositories if you've already initialized locally
# If not, create a new repository on GitHub and then run the following commands:
git remote add origin <repository_URL>
git branch -M main
git push -u origin main

2. GitHub Edit Code and Commit Changes:

# Step 1: Make changes to your local code
# For example, open a file in a text editor and make edits

# Step 2: Add changes to the staging area and commit
git add .
git commit -m "Description of changes"

# Step 3: Push changes to GitHub
git push

3. Git Pull, Fetch and Merge:

# Step 1: Fetch changes from the remote repository (GitHub)
git fetch origin

# Step 2: Merge fetched changes into your local branch
git merge origin/main

# Alternatively, you can combine fetch and merge in one command
git pull origin main

4. Git Push:

# Push local changes to the remote repository (GitHub)
git push

5. GitHub Branch:

# Step 1: Create a new branch locally
git branch feature-branch
git checkout feature-branch
# Or use the shorthand for the above two commands
# git checkout -b feature-branch

# Step 2: Make changes and commit to the new branch
git add .
git commit -m "Changes in feature branch"

# Step 3: Push the new branch to GitHub
git push -u origin feature-branch
